## 项目经历

### 客服指令模型训练与部署平台

**技术栈**：Python、Qwen2.5-7B-Instruct、4bit QLoRA、LLaMA-Factory、vLLM、LCS

核心目标：解决电商客服意图识别、流程路由和槽位抽取场景中，大模型输出不稳定、难以被业务系统解析的问题。

- 设计客服指令 SFT 样本格式，覆盖订单查询、物流查询、售后申请及负样本拒识场景，使模型输出可被业务系统解析的标准化命令。
- 开发数据构造、增强、清洗与划分脚本，基于 DeepSeek 生成多样化中文客服问法，形成 1000 条原始样本和 1993 条增强训练样本。
- 采用 Qwen2.5-7B-Instruct 作为基座模型，使用 4bit QLoRA 完成指令微调，并通过 LLaMA-Factory 管理训练配置与 Adapter。
- 开发多维度评估脚本，统计 Flow Accuracy、Slot F1、格式合法率和负样本拒识率；在 201 条测试样本上 Flow Accuracy 达 98.50%，格式合法率与负样本拒识率均为 100%。
- 使用 vLLM 部署微调模型并暴露 OpenAI 兼容接口，接入 LCS 业务系统，形成“数据生成-微调-评估-部署”的完整闭环。

核心目标：解决企业产品手册、设备说明书等非结构化文档检索效率低、用户难以快速定位答案的问题。

- 构建查询侧 RAG 工作流，整合上下文改写、HyDE 检索、RRF 融合和 BGE Reranker 精排，提升答案生成前的上下文质量。
- 接入 MongoDB 保存会话历史与处理状态，结合 FastAPI / SSE 返回处理进度和流式回答；使用 MinIO 管理文档图片资源。
